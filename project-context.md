Project Context: Full-Stack ToDo App (Nuxt 3 + MSW)
🎯 Обзор проекта

Разработка приложения для управления задачами с авторизацией, CRUD-операциями, фильтрацией и сортировкой. Проект выполняется как тестовое задание для Frontend Developer.
🔗 Репозиторий проекта

    GitHub: https://github.com/a197428/todo-app

    Текущее состояние: В репозитории инициализирован Nuxt 3 проект с конфигурацией srcDir: 'src', настроен Tailwind CSS и интегрирована папка msw-mocks для имитации бэкенда.

🛠 Технологический стек

    Framework: Nuxt 3 (SSR: disabled).

    Styling: Tailwind CSS.

    Language: TypeScript (Strict mode).

    API Mocking: MSW v1.x (Legacy).

    Environment: Ubuntu Linux, Vite.

🧪 Тестирование (Важное требование)

Необходимо настроить и реализовать следующие уровни тестирования:

    Unit-тесты: Тестирование вспомогательных функций (utils) и composables (например, логика валидации, форматирование дат).

    Component Testing: Проверка рендеринга и поведения ключевых компонентов (TaskCard, AuthForm).

    E2E / Integration Testing: Проверка основных сценариев: "Авторизация -> Создание задачи -> Редактирование -> Удаление".

        Инструменты: Рекомендуется использовать Vitest для Unit/Component тестов и Playwright или Cypress для E2E.

        Важно: Тесты должны учитывать наличие MSW и корректно работать с перехваченными запросами.

📋 Техническое задание (Full Requirements)

1. Авторизация и безопасность

   Форма: Email/Password.

   Token: Хранение JWT в localStorage.

   Security: Автоматический перехват 401 ошибок, очистка токена и редирект на /login.

2. Управление задачами (CRUD)

   Модель Task: Id, Title, Description, DueDate, IsCompleted, CreatedBy.

   Чтение: Список задач с индикацией загрузки (спиннеры).

   Создание: Форма с валидацией (название, описание, дедлайн, статус).

   Изменение/Удаление: Через модальные окна или inline-режим.

3. Сортировка, Фильтрация и Поиск

   Сортировка: По дате создания, дедлайну и статусу.

   Фильтры: «Только активные», «Выполненные», «Просроченные».

   Поиск: С использованием debounce и обработкой состояния «результаты не найдены».

4. Дополнительный функционал

   Роли: Разделение на Admin и User.

   Права: Редактирование/удаление доступно только создателю задачи или админу.

   UX: Пагинация списка и информативные сообщения об ошибках API.

📡 API Эндпоинты (MSW v1)

    POST /api/auth/login — Авторизация.

    GET /api/tasks — Получить список задач.

    POST /api/tasks — Создать задачу.

    PUT /api/tasks/:id — Обновить задачу.

    DELETE /api/tasks/:id — Удалить задачу.

⚠️ Инструкции для ИИ-ассистента

    Strict Typing: Импортируй типы только как import type из ~/msw-mocks/data/tasks.data.

    Testing Setup: Сгенерируй конфигурацию для Vitest и напиши примеры тестов для useApi и основных компонентов.

    Vite Compatibility: Учитывай, что проект использует Vite. При настройке тестов убедись в совместимости с Nuxt-окружением.

    MSW Usage: Не обновляй MSW до v2. Используй rest из msw.
